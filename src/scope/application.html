APPLICATION.JS

Here it is defined the ApplicationScope or EVT.App. EVT.App is a sub-class of scope and it defines the public API that an App Api Key can access to.

An Application scope currently has access to:

  • Product resource (R)
  • Action resource (C) - Scans only
  • App User resource (C)
  • Login
  • (C actions via products)

Implement Public API by extending the prototype.

By default all resource constructors are themselves factory functions that are called by the scopes, can receive an ID and return a Resource. However, in some situations in our API, the output of different endpoints can return be the same. Thus we need to setup the resource constructor to use a certain path, and return the correct factory function. This is what happens here with the appUser() resource constructor.
